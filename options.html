Gmail GitHub PR Status

Paste a GitHub token so the extension can read issue and PR status. A token is required for private repositories (e.g. your neeto repos).

How to create a token

Go to github.com/settings/tokens → Generate new token → Fine-grained, then:

  1. Token name: Gmail GitHub status for PRs
  2. Expiration: No expiration
  3. Resource owner: pick neetozone here. This dropdown is how you add the org — there is no separate “add org” prompt. If neetozone is not listed, the org has not enabled fine-grained tokens (or needs admin approval) — use the classic token below instead.
  4. Repository access: All repositories (or just the ones you get emails about).
  5. Repository permissions: Issues → Read-only and Pull requests → Read-only.
  6. Generate token, copy it, paste it above, and click Save.

Seeing “No repo access” or “Not found”? Your token is valid but can’t read the private repo — the fine-grained token was most likely created under your personal account instead of neetozone. Easiest fix: create a classic token with the repo scope. It works across every org you belong to with no per-org opt-in or approval. (The Test button only checks that the token is valid, not that it can see your repos.)

The token is stored only in this browser via chrome.storage.local and is sent only to api.github.com.
